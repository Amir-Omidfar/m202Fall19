App - Server Process Flow

The android based application enables speech to text feature and let user ask for specific destination he desires to go. The other command this feature allows user to send is to adding new location to the list of current destination to which DirectMe can guide the user. The CGI (Common Gate Interface) server receives the user's command. If the command has any of the words given in pre-defined "places.txt" file, it caculates the shortest path and send step by step instructions to guide the user. If the command received has the "add" it then takes the current position of the user and assigns it to the name it asks the user to say. The position and name of the new location is then added to places.txt file.

The shortest path calculation

The shortest path algorithm we applied uses a visibility graph. Our pre defined four nodes are around the conference table which is the main obstacle for our case. The start and goal position are the user's position at the start of navigation and the destination he specifies. The edges along this visibility graph are assigned in the way that they are not going through the obstacle (the conference table). From there the algorithm canculates the path the user needs to take. It then updates its commands as user walks towards the destination.

The Application interface

As shown in the picture, the interface is mainly voice command based. The two basic commands are "adding new location" by saying "add ..." and declaring the destination the user wants to go to. The command "Pause" let the user stops the navigation process and in order to resume he may double tap the screen.These three commnads are in the main menu of the app and that is where the user start his interaction with the app. The navigation commands are played to the user in explicit way of how many meters he needs to walk to how many degrees (Clockwise or Counter Clockwise) he needs to turn. While commands are played to the user from his smart phone. Tapping twice on the screen would take him to the command-only menu. There he would again have acces to "Add" and "Pause" commnads along with "Return" and "Cancel". Now with the "Return" command, it would go back to navigation mode (playing path guiding commands) and whether the user says "Cancel" it would stop the navigation process going back to the main menu.

The Bluetooth Beacon

As measured during the testing process, UWB positioning system has about 20-30 cm error rate. In order to make up for that, while in 20-40 cm proximity of the destination, we uses a Bluetooth beacon (placed on the target appliance/destination). Proportional to the signal strength received from the beacon, the smart phone (ideally placed on the user's wrist) vibrates. The closer to the destination it gets, the more vibration the phone gives off. This feature let the user search and find the destination when he gets within 20-40 cm range.
